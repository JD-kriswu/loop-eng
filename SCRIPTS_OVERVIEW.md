# Loopany 项目脚本总览

本项目提供了一套完整的运维脚本，解决之前手动重启时遇到的所有问题。

## 🚀 快速开始

```bash
# 重启服务（推荐使用）
./restart.sh

# 查看状态
./status.sh

# 停止服务
./stop.sh

# 验证删除功能
./verify_delete.sh http://localhost:3001
```

## 📁 脚本文件

| 脚本 | 功能 | 使用场景 |
|------|------|----------|
| `restart.sh` | 编译并重启服务 | 代码修改后重启 |
| `stop.sh` | 停止服务 | 手动停止服务 |
| `status.sh` | 检查服务状态 | 验证服务是否正常 |
| `verify_delete.sh` | 验证删除功能 | 测试硬删除是否工作 |

## 🔧 之前遇到的问题（已解决）

### 问题 1: 端口看错
- ❌ 之前：总是查看 3000 端口
- ✅ 现在：脚本默认使用 3001 端口，可通过参数指定

### 问题 2: 进程未正确停止
- ❌ 之前：手动 kill 经常失败
- ✅ 现在：自动查找进程，优雅停止 + 强制停止

### 问题 3: 数据库连接错误
- ❌ 之前：忘记设置 DATABASE_URL
- ✅ 现在：脚本自动设置正确的数据库连接

### 问题 4: 日志目录不存在
- ❌ 之前：nohup 失败因为 logs 目录不存在
- ✅ 现在：脚本自动创建日志目录

### 问题 5: 启动成功但无验证
- ❌ 之前：启动后没有验证是否真正工作
- ✅ 现在：自动验证进程、端口、API 响应

## 🎯 核心优势

### 1. 防止人为错误
- 自动处理所有依赖（数据库、日志目录）
- 正确的端口配置
- 完整的验证流程

### 2. 可复用性
- 一次编写，持续使用
- 团队成员都可以使用相同脚本
- 避免每个人都要记住复杂的重启步骤

### 3. 可观测性
- 清晰的输出信息
- 明确的成功/失败状态
- 快速命令提示

### 4. 健壮性
- 处理进程不响应的情况
- 完整的错误检查
- 清晰的失败原因提示
